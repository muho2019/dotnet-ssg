# 🚀 dotnet-ssg 로드맵

경쟁력 있는 오픈소스 정적 사이트 생성기가 되기 위한 분석 및 로드맵 문서입니다.

---

## ✅ 현재 갖춘 기능

- 마크다운 → HTML 변환 (Markdig)
- Front Matter 파싱 (YamlDotNet)
- Blazor 컴포넌트 기반 템플릿
- Tailwind CSS 통합
- SEO 기본 지원 (sitemap, robots.txt, RSS)
- 태그 아카이브

---

## 🔴 필수로 추가해야 할 기능

| 영역                | 필요 기능                                                      | 이유                             |
| ------------------- | -------------------------------------------------------------- | -------------------------------- |
| **CLI 도구**        | `dotnet tool` 패키지화, `init`, `new`, `serve`, `build` 명령어 | 사용자가 쉽게 설치하고 사용 가능 |
| **개발 서버**       | Hot Reload 지원 로컬 서버                                      | 실시간 미리보기 필수             |
| **페이지네이션**    | 포스트 목록 페이지 분할                                        | 대량 콘텐츠 처리                 |
| **드래프트 지원**   | `draft: true` 플래그로 미공개 포스트 관리                      | 표준 기능                        |
| **카테고리/시리즈** | 태그 외에 카테고리, 시리즈 지원                                | 콘텐츠 구조화                    |
| **테마 시스템**     | 교체 가능한 테마 구조                                          | 커스터마이징 용이성              |
| **다국어(i18n)**    | 다국어 콘텐츠 관리                                             | 글로벌 사용자 대응               |
| **증분 빌드**       | 변경된 파일만 재빌드                                           | 대규모 사이트 빌드 속도          |

## 🟡 권장 추가 기능

| 영역                    | 필요 기능                                     | 이유             |
| ----------------------- | --------------------------------------------- | ---------------- |
| **검색 기능**           | 클라이언트 사이드 검색 (Lunr.js, Pagefind 등) | UX 향상          |
| **이미지 최적화**       | 자동 리사이즈, WebP 변환, lazy loading        | 성능 최적화      |

| **목차(TOC) 자동 생성** | 헤딩 기반 Table of Contents                   | 긴 문서 탐색     |
| **Related Posts**       | 관련 포스트 자동 추천                         | 사용자 참여 증가 |
| **Syntax Highlighting** | 코드 블록 하이라이팅 테마 선택                | 기술 블로그 필수 |
| **Asset Pipeline**      | JS/CSS 번들링, 미니파이                       | 성능 최적화      |
| **데이터 파일**         | JSON/YAML 데이터 파일 로드                    | 동적 콘텐츠 생성 |
| **환경별 설정**         | dev/prod 설정 분리                            | 개발 편의성      |

## 📊 Hugo/Jekyll/Eleventy 대비 차별화 포인트

1. **Blazor 컴포넌트** - .NET 생태계에서 유일한 Razor 기반 SSG
2. **C# 확장성** - 플러그인을 C#으로 작성 가능
3. **.NET 개발자 친화적** - 기존 .NET 프로젝트와 쉬운 통합
4. **Aspire 통합** - .NET Aspire와 연동한 현대적 개발 경험
