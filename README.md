# dotnet-ssg: .NET 기반 정적 사이트 생성기

이 프로젝트는 .NET을 사용하여 개인 블로그를 만들기 위한 간단하고 효율적인 정적 사이트 생성기(Static Site Generator)입니다.

Gemini CLI와 함께 개발합니다.

## 🌟 프로젝트 목표

마크다운(`.md`) 파일로 콘텐츠를 작성하고, Scriban 템플릿을 통해 커스터마이징된 디자인의 개인 블로그를 생성하는 것을 목표로 합니다.

## ✨ 주요 기능

- **마크다운 기반 콘텐츠**: 모든 포스트와 페이지는 간단한 마크다운 파일로 관리됩니다.
- **Scriban 템플릿 엔진**: Liquid 기반의 Scriban을 사용하여 유연하고 강력한 템플릿을 작성할 수 있습니다.
- **SEO 최적화**: 검색 엔진에 친화적인 메타데이터와 페이지 구조를 자동으로 생성합니다.
- **빠른 빌드 속도**: 콘텐츠 파싱 및 렌더링 과정을 병렬로 처리하여 빠른 빌드를 지원합니다.
- **Tailwind CSS 지원**: 유틸리티 우선 CSS 프레임워크인 Tailwind CSS를 사용하여 현대적이고 반응형인 디자인을 쉽게 적용할 수 있습니다.

## 🛠️ 기술 스택

- **프레임워크**: .NET 10
- **마크다운 파싱**: Markdig
- **Front Matter 파싱**: YamlDotNet
- **템플릿 엔진**: Scriban
- **CSS 프레임워크**: Tailwind CSS
- **설정 관리**: System.Text.Json

## 🚀 시작하기

1.  **콘텐츠 작성**:
    *   블로그 게시물은 `content/posts` 디렉토리에 마크다운 파일(`.md`)로 작성합니다.
    *   일반 페이지(예: About, Contact)는 `content/pages` 디렉토리에 마크다운 파일(`.md`)로 작성합니다.
    *   CSS, JavaScript, 이미지 등 정적 파일은 `content/static` 디렉토리에 배치합니다. 이 파일들은 빌드 시 `output/static`으로 복사됩니다.
2.  **사이트 빌드**:
    프로젝트 루트 디렉토리에서 다음 명령어를 실행하여 정적 사이트를 생성합니다:
    ```bash
    dotnet run --project src/DotnetSsg
    ```
3.  **배포**:
    생성된 정적 파일은 `output` 디렉토리에 있습니다. 이 파일들을 원하는 웹 서버에 배포하십시오.
